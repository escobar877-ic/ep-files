# Архитектурные паттерны бэкенда EP Files

## 1. Repository Pattern (Репозиторий)
Используется для изоляции логики работы с базой данных. Вся работа с ORM Django (запросы, создание записей) выносится в папку repositories.

## 2. Service Layer (Сервисный слой)
Здесь живет бизнес-логика: проверка прав доступа, валидация лимитов на размер файла, вызов сторонних API. Контроллеры (api/views) только вызывают методы из этого слоя.

## 3. Configuration Object (Объект конфигурации)
Все константы (лимиты, пути к папкам, секреты) вынесены в отдельный модуль core/config.py. Это избавляет от жестко прописанных значений (hardcode) по всему проекту.
