# Архитектурные паттерны бэкенда EP Files

## 1. Repository Pattern (Репозиторий)
Место: ep_files_app/repositories/
Используется для изоляции логики работы с базой данных. Вся работа с ORM Django выносится в репозитории, чтобы не загромождать вьюхи.

## 2. Service Layer (Сервисный слой)
Место: ep_files_app/services/
Здесь живет бизнес-логика: проверка прав доступа, валидация лимитов. Файл file_service.py реализует логику обработки файлов перед сохранением.

## 3. Configuration Object (Объект конфигурации)
Место: ep_files_app/core/config.py
Все константы (лимиты, пути) вынесены в отдельный модуль. Это избавляет от хардкода по всему проекту.
