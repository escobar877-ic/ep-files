# Правила работы с репозиторием (Git Flow)

## Ветки
- `main` — стабильная версия для продакшена (только merge через MR).
- `dev` — рабочая ветка для разработки (merge из feature/*).
- `feature/название-задачи` — новые фичи (от dev).
- `bugfix/описание` — исправления багов (от dev).
- `hotfix/версия` — срочные фиксы в main.

## Правила коммитов
- Формат: `тип: описание` (feat:, fix:, docs:, test:, chore:).
- Английский или русский — как удобно.
- Не больше 72 символов в строке.

## Workflow
1. `git checkout dev && git pull`
2. `git checkout -b feature/твоя-задача`
3. Код + коммиты.
4. `git push origin feature/твоя-задача`
5. Создай MR в `dev` с ревью от 1 человека.
