# ✅ Проблема решена!

## Что было не так?

У вас была ошибка **404 (Page not found)** при открытии корневого URL `http://127.0.0.1:8000/`.

### Причина
В файле `main/urls.py` не было маршрута для корневого пути `/`. Все API endpoints были доступны только через `/api/`, но Django не знал, что показывать на главной странице.

## Что было исправлено?

### 1. Добавлен корневой маршрут
В `main/urls.py` добавлена функция `home_view()`, которая возвращает JSON с информацией об API:

```python
def home_view(request):
    """Главная страница API"""
    return JsonResponse({
        'message': 'EP-Files API',
        'version': '1.0',
        'endpoints': {
            'admin': '/admin/',
            'api': '/api/',
            'auth': {...},
            'files': {...},
            'frontend': 'http://localhost:5173'
        }
    })
```

### 2. Улучшен скрипт запуска
Файл `run.sh` теперь:
- Останавливает существующие процессы перед запуском
- Проверяет успешность запуска Django
- Показывает полезные ссылки
- Корректно останавливает серверы при Ctrl+C

## Как использовать проект?

### Запуск проекта
```bash
./run.sh
```

### Доступные URL:
- **Frontend**: http://localhost:5173
- **Backend API**: http://localhost:8000
- **API Documentation**: http://localhost:8000/ (список всех endpoints)
- **Admin Panel**: http://localhost:8000/admin/

### Основные API endpoints:

#### Аутентификация
- `POST /api/auth/register/` - Регистрация
- `POST /api/auth/login/` - Вход
- `GET /api/auth/me/` - Информация о текущем пользователе

#### Файлы
- `POST /api/upload/` - Загрузка файла
- `GET /api/files/` - Список файлов
- `GET /api/files/<id>/download/` - Скачать файл
- `DELETE /api/files/<id>/` - Удалить файл

#### Папки
- `GET /api/folders/` - Дерево папок
- `POST /api/folders/create/` - Создать папку
- `PUT /api/folders/<id>/rename/` - Переименовать папку
- `DELETE /api/folders/<id>/delete/` - Удалить папку

#### Права доступа
- `POST /api/files/<id>/permissions/grant/` - Выдать права на файл
- `POST /api/files/<id>/permissions/revoke/` - Отозвать права
- `GET /api/permissions/my/` - Мои права доступа

### Остановка проекта
Нажмите `Ctrl+C` в терминале, где запущен `run.sh`

Или вручную:
```bash
pkill -f "manage.py runserver"
pkill -f "vite"
```

## Проверка работы

```bash
curl http://127.0.0.1:8000/
```

Должен вернуть JSON с информацией об API.

### Frontend
Откройте в браузере: http://localhost:5173

## Дополнительная информация

- Проект использует **Django** (backend) и **React + Vite** (frontend)
- База данных: **SQLite** (файл `db.sqlite3`)
- Аутентификация: **JWT токены**
- Файлы хранятся в папке `storage/files/`

## Если что-то не работает

1. Убедитесь, что виртуальное окружение активировано:
   ```bash
   source venv/bin/activate
   ```

2. Проверьте, что все зависимости установлены:
   ```bash
   pip install -r requirements.txt
   cd frontend && npm install
   ```

3. Примените миграции:
   ```bash
   python manage.py migrate
   ```

4. Проверьте логи в `logs/app.log`
