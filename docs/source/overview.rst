Project overview
================

EP-Files состоит из backend-части на Django и Django REST Framework и frontend-части
на React. Backend отвечает за хранение пользователей, файлов, папок, прав доступа,
публичных ссылок, истории действий и административных операций.

Основные части backend
----------------------

* ``ep_files_app.api`` — HTTP API для авторизации, файлов, папок, прав доступа,
  публичных ссылок, истории, избранного и админ-панели.
* ``ep_files_app.models`` — модели пользователей, файлов, папок, истории и прав.
* ``ep_files_app.services`` — бизнес-логика, вынесенная из API-представлений.
* ``ep_files_app.permissions`` — классы проверки прав доступа DRF.
* ``ep_files_app.middleware`` — middleware для безопасности и проверки запросов.
* ``ep_files_app.observers`` — реализация наблюдателей для событий файлов.
* ``ep_files_app.validators`` — проверки имени, размера, расширения и MIME-типа файла.

Почему проект разделён на модули
--------------------------------

Изначально часть логики находилась в крупных файлах. После рефакторинга код был
разделён по зонам ответственности: файлы, папки, права доступа, публичные ссылки,
текстовый редактор, администрирование и история действий. Это упрощает поддержку
проекта, поиск нужной логики и объяснение кода на защите.
